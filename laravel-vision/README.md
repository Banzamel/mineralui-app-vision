# laravel-vision

Laravel 12 modular monolith powering the Vision API. Multi-tenant by `company_id`, OAuth via Passport, RBAC via Spatie Permission (teams mode), realtime broadcasting via Reverb.

## Domain contract

Every business area is a domain under `src/Domains/{Domain}` and **must** keep the same 10 top-level folders, even when some are empty (use `.gitkeep`):

```
Dtos/  Enums/  Events/  Factories/  Models/  Observers/  Repositories/  Requests/  Resources/  Services/
```

| Folder         | Responsibility                                                                                |
|----------------|-----------------------------------------------------------------------------------------------|
| `Dtos`         | Immutable input/output objects passed between requests, controllers and services.             |
| `Enums`        | Domain enums and strongly typed value sets.                                                   |
| `Events`       | Domain events and listeners (`Events/Listeners/`).                                            |
| `Factories`    | Eloquent factories.                                                                           |
| `Models`       | Eloquent models and relations.                                                                |
| `Observers`    | Model lifecycle rules тАФ kept out of services and controllers.                                 |
| `Repositories` | Data access. Concrete classes here, contracts in `Repositories/Interfaces/`.                  |
| `Requests`     | `FormRequest` validation + DTO mapping.                                                       |
| `Resources`    | JSON serializers.                                                                             |
| `Services`     | Business logic. Concrete classes here, contracts in `Services/Interfaces/`.                   |

`src/Shared/` holds cross-domain helpers (traits, scopes, exceptions, middleware) and is **not** a domain.

## Request flow

```
Route -> Middleware -> FormRequest -> Controller (__invoke) -> Service -> Repository -> Model / Observer -> Event / Resource
```

- Controllers are single-action and stay thin (`__invoke` only).
- Requests validate input and produce a DTO via `getDto()`.
- Services orchestrate business logic and call repositories.
- Repositories hold all query / persistence code тАФ services never use Eloquent directly.
- Resources shape the response; collection responses must use `Resource::collection($x)->response()` so the `{data: [...]}` envelope stays.

DI bindings (interface тЖТ concrete) live centrally in [`src/App/Providers/RegisterServiceProvider.php`](src/App/Providers/RegisterServiceProvider.php).

## Default domains

- **Administration** тАФ users, companies, roles, permissions, avatars.
- **Auth** тАФ login, logout, refresh, social login, auth logs.
- **Installer** тАФ `/install` wizard backend (database test, admin, first object, first camera, finalize).
- **Objects** тАФ Vision tree (buildings/apartments/тАж), cameras, scopes.
- **Albums** тАФ daily albums + photo cursor pagination + retention.
- **FileManager** тАФ directory tree under `storage/app/private/{company_id}/...`, mirrored against the object tree by `CameraObserver`.
- **Notifications** тАФ in-app notifications + Web Push (VAPID, `minishlink/web-push`). Payloads carry a structured `data` field (e.g. `{actor_name}`, `{date, camera_name, album_id}`) so the frontend renders titles/messages through i18n; `title`/`message` columns stay populated with EN fallback for OS-level Web Push and graceful UI degradation.

## Routing

- [`routes/oauth.php`](routes/oauth.php) тАФ public auth endpoints.
- [`routes/api.php`](routes/api.php) тАФ protected endpoints, grouped by `manage/*`, `administration/*`, `files/*`, plus install routes (gated by `install.gate`).

## Auth & multi-tenancy

- **Passport** for OAuth2 (password grant, public client `Vision Web` created during install).
- **Spatie Permission** in teams mode keyed on `company_id`. Permissions defined in [`config/permission.php`](config/permission.php).
- **Global scope** `BelongsToCompany` filters every tenant table by the authed user's `company_id`.
- `Request::authorize()` gates every action with `$this->user()?->can('x') === true`.

## Install state

`config('app.installed')` is the single source of truth, fed by `APP_INSTALLED` in `.env`. The installer wizard flips it via `EnvWriter` on finalize. No JSON state files.

## Storage layout

Two disks, two flavours of photo storage:

- **Private** (`storage/app/private/`, default disk) тАФ camera footage, organised by FileManager. Path: `{company_id}/{root_object_slug}/.../{camera_slug}/{day}/photo.jpg`. Each day-folder gets a sibling `thumbs/{filename}.jpg` (400├Ч300 JPEG q80) generated by `Albums\Services\ThumbnailService` тАФ synchronously on first view, async via the `GenerateThumbnailListener` queued on `PhotoAddedEvent`. `CameraObserver` walks the `VisionObject` ancestor chain on camera creation and provisions the directory chain.
- **Public** (`storage/app/public/`, served via the `public/storage` symlink) тАФ main photos for objects (`object-photos/{company_id}/{id}/`) and cameras (`camera-photos/{company_id}/{id}/`), plus user avatars (`avatars/{company_id}/`). Uploaded through dedicated endpoints (`POST /api/vision/objects/{id}/main-photo`, `.../cameras/{id}/main-photo`); models expose `getMainPhotoUrlAttribute()` so resources return a browser-loadable URL.

## Photo URLs and auth

`<img>` tags can't send a Bearer token, so photo streaming dodges `auth:api`:

- `Albums\Resources\PhotoResource` issues short-lived **signed URLs** via `URL::temporarySignedRoute(..., absolute: false)` for both the full stream (`vision.photos.stream`) and the thumbnail (`vision.photos.thumb`).
- Both routes sit under `Route::middleware('signed:relative')` at the bottom of `routes/api.php` тАФ outside the `auth:api` group. Tampering with `id` invalidates the HMAC signature.
- Public-disk URLs (avatars, main photos) skip signing entirely тАФ they're served from the `public/storage` symlink with no Laravel routing.

## Local commands

Run inside the container (`docker compose exec laravel-vision bash`) or directly if you have PHP 8.4 on the host:

```bash
composer install
php artisan migrate
composer test
```

Tests live under `tests/Unit` (services, listeners, policies тАФ currently 145/145 green).

## Production deploy

Production runs on HestiaCP (single domain, no Docker): Apache+nginx hybrid, PHP-FPM 8.4 per-domain socket, Supervisor for queue workers + Reverb. Build artifacts and deploy scripts live in `prod/` (gitignored тАФ generated by `prod/build.sh`).
